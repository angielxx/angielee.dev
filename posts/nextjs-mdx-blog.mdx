## 왜 MDX인가

MDX는 Markdown에서 JSX를 사용할 수 있게 해주는 포맷이다. 글을 마크다운으로 편하게 쓰면서도, 필요할 때 React 컴포넌트를 직접 삽입할 수 있다.

CMS를 쓰지 않아도 로컬 파일로 모든 글을 관리할 수 있다는 것도 큰 장점이다.

## 프로젝트 구조

```
portfolio/
├── app/
│   └── blog/
│       └── [slug]/
│           └── page.tsx
├── lib/
│   └── posts.ts
└── posts/
    ├── hello-world.mdx
    └── nextjs-app-router.mdx
```

## gray-matter로 frontmatter 파싱

각 MDX 파일 상단의 `---` 블록이 frontmatter다.

```ts
import matter from "gray-matter";
import fs from "fs";

const raw = fs.readFileSync("posts/hello-world.mdx", "utf-8");
const { data, content } = matter(raw);

// data: { title, date, category, tags, ... }
// content: 본문 MDX 문자열
```

### frontmatter 구조

```yaml
---
title: "글 제목"
date: "2026-02-26"
description: "설명"
category: "프론트엔드"
tags: ["Next.js", "MDX"]
series: "시리즈 이름"
seriesOrder: 1
---
```

## next-mdx-remote로 렌더링

`next-mdx-remote/rsc`는 Server Component에서 바로 사용할 수 있다.

```tsx
import { MDXRemote } from "next-mdx-remote/rsc";
import remarkGfm from "remark-gfm";
import rehypeSlug from "rehype-slug";

export default async function BlogPost({ params }) {
  const post = getPostBySlug(params.slug);

  return (
    <article className="prose">
      <MDXRemote
        source={post.content}
        options={{
          mdxOptions: {
            remarkPlugins: [remarkGfm],
            rehypePlugins: [rehypeSlug],
          },
        }}
      />
    </article>
  );
}
```

### rehype-slug와 TOC

`rehype-slug`는 헤딩에 자동으로 id를 부여한다.

```html
<!-- 변환 전 -->
<h2>App Router란</h2>

<!-- 변환 후 -->
<h2 id="app-router란">App Router란</h2>
```

TOC에서 이 id를 앵커 링크로 활용하면 클릭 시 해당 섹션으로 스크롤된다.

## 읽기 시간 계산

한국어 기준 분당 약 500자를 읽는다고 가정한다.

```ts
export function calcReadingTime(content: string): number {
  const text = content
    .replace(/^---[\s\S]*?---/, "")  // frontmatter 제거
    .replace(/```[\s\S]*?```/gm, "") // 코드블록 제거
    .replace(/[#*`\[\]()>_~]/g, "")  // 마크다운 문법 제거
    .trim();
  return Math.max(1, Math.ceil(text.length / 500));
}
```

## 마무리

gray-matter + next-mdx-remote 조합으로 외부 CMS 없이 완전한 블로그를 만들 수 있다. 파일 기반이라 Git으로 버전 관리가 되고, 로컬에서 바로 글을 쓸 수 있다는 점이 개발자에게 잘 맞는다.
